------------------------------------------------------ocx 注册与注册表修改-------------------------------------------------------
这是一个64位的activeX控件，非常特殊

在64位和32位的系统下，有不同的方式

在64位系统下，System32和SysWow64文件夹下面有两个注册的exe，regsvr32.exe分别用来注册不同位的ocx，/u是去除注册
64位系统下，system32里都是64位的，而syswow64里则是处理32位的。

然后，ie浏览器也分为32位和64位，当然32位只能加载32位的ocx，64位的可以加载32和64位的ocx
问题是64位的ie即使运行后，标签页也是32位的，这时候的解决方法根据系统不同而不同

win7下，可以选择修改注册表的方式：
Windows Registry Editor Version 5.00    
[HKEY_CURRENT_USER\Software\Microsoft\Internet Explorer\Main]
"TabProcGrowth"=dword:00000000

win10下，首先通过组策略修改ie运行方式为单进程方式，然后才能进行下一步操作
组策略->管理模板->所有设置->设置标签页进程增长，编辑，已启用，标签页进程增长设为0
再通过修改注册表，达到和win7一样的效果。
